# Security policy

This project executes code and writes files in target repositories. Reports should identify the affected command, operating system, Python version, and a minimal reproduction without including credentials or private repository content.

Security-sensitive invariants:

- Resolve all input and output paths against the repository root.
- Reject writes outside the repository, through `.git`, or through escaping symbolic links.
- Never execute a mutable Git reference supplied by a generated bootstrap file.
- Do not silently overwrite files that are not marked as generated by this tool.
